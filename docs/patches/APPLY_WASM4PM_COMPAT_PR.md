## PR Title

`feat: Add Oracle class witness markers A8-A12 for anti-LLM cheat detection`

## PR Body

### Summary

Adds structural foundation for `lsp-max-anti-cheat` to classify LLM coding
agent cheating patterns via OCEL-based process mining (Oracle classes A8–A12).

**New file: `src/witnesses_anti_cheat.rs`**

Seven witness markers — all structure-only, all zero-cost uninhabited enums:

| Marker | Oracle | What it names |
|--------|--------|---------------|
| `OracleA8AuditLogTampering` | A8 | Receipts without ancestor computation events |
| `OracleA9TemporalAnomaly` | A9 | Events with impossible timestamps |
| `OracleA10CausalViolation` | A10 | Mutations/diagnostics without causal predecessors |
| `OracleA11UnknownCollapse` | A11 | UNKNOWN → ADMITTED without resolution event |
| `OracleA12CyclicDependency` | A12 | Cycles in the OCEL event causality DAG |
| `LspMaxSessionWitness` | — | Authority label for lsp-max session OCEL events |
| `AntiLlmScanWitness` | — | Authority label for anti-llm-cheat scanner OCEL events |

**Updated: `src/diagnostic.rs`**

Added five `CompatDiagnostic` variants (`OracleA8` through `OracleA12`) with
law descriptions and `Display` messages. These let lint gates and CI report
exactly which Oracle class a boundary violation maps to.

**Updated: `src/lib.rs`**

Registered `witnesses_anti_cheat` module. Marked as hand-authored (not
ggen-generated) since it documents the lsp-max threat model boundary.

### Design Invariants Preserved

- **Structure only**: zero engines, zero detection logic. Oracle detection
  belongs in `wasm4pm`; this crate only names the authority classes.
- **No forbidden language**: no `legacy`, `deprecated`, `shim`, `facade`, or
  `backward compatibility` in any added file.
- **No intermediary types**: no new `ocel_core` or `wasm4pm_types`. Uses
  existing `wasm4pm-compat` type vocabulary throughout.
- **Witnesses travel with evidence**: `oracle_class` KEY strings flow through
  `Evidence<T, State, W>` and `Admission<T, W>` without runtime cost.

### Consumer: lsp-max

`lsp-max/crates/lsp-max-anti-cheat/src/rules/ocel_rules.rs` emits Oracle
class KEY strings (`anti-cheat/oracle-a8-audit-log-tampering`, etc.) on all
seven OCEL violation rules (OCEL-001 through OCEL-007). The `AntiLlmDiagnostic`
struct carries `oracle_class: Option<String>` and `confidence: Option<f64>`.
